## 質問と回答の記録
Q1

質問：
@​State は何のために使うのですか？

AIの回答の要点：
@​State は、画面の中で変わる値を保存するために使う。
その値が変わると、SwiftUI は画面を自動で更新する。

自分の理解：
@​State は、地図の位置や選択状態のような変化する情報を持つために必要だと分かった。
これがないと、値を変えても画面に反映されにくい。

Q2

質問：
なぜ Landmark に Identifiable をつけるのですか？

AIの回答の要点：
Identifiable をつけると、それぞれのデータを区別できる。
For​Each で一覧表示するときに必要になることが多い。

自分の理解：
地図上に複数の場所を表示するとき、SwiftUI がどのデータか判断しやすくなるため必要だと分かった。
1つ1つの場所を正しく扱うための仕組みだと思った。

Q3

質問：
Set を使ってカテゴリを管理する理由は何ですか？

AIの回答の要点：
Set は同じ値を重複して入れない。
追加と削除がしやすいので、選択中のカテゴリを管理するのに向いている。

自分の理解：
カテゴリのオン・オフを切り替える処理に合っていると分かった。
配列よりも、選ばれているかどうかを簡単に確認できるのが便利だと思った。

Q4

質問：
Map(position:) の position には何を入れているのですか？

AIの回答の要点：
position には、地図の中心や表示範囲などの情報を入れる。
これによって、最初にどこを表示するかを決められる。

自分の理解：
地図の見せ方を決める大事な値だと分かった。
東京駅付近を最初に表示するために使っているのだと理解した。
Q5

質問：
Marker は何をするものですか？

AIの回答の要点：
Marker は、地図の上に場所の目印を表示するための部品である。
名前やアイコンを一緒に表示できる。

自分の理解：
場所を見つけやすくするための印だと分かった。
これを使うことで、ランドマークの位置がすぐ分かるようになる。
